Aufgabe 8 – Batch Processing

Viele INSERTs effizient ausführen

Aufgabe:

Erstelle ein Java-Programm, das viele Personen in eine H2-Datenbank einfügt.

Das Programm soll zwei Varianten vergleichen:

  • Einfügen mit vielen einzelnen executeUpdate()-Aufrufen
  • Einfügen mit addBatch() und executeBatch()

Miss für beide Varianten die Ausführungszeit und gib die Ergebnisse auf der Konsole aus.

So soll sichtbar werden, dass Batch Processing bei vielen Datensätzen effizienter sein kann.

Schritt 1: Properties-Datei verwenden

Verwende wieder eine Datei connection.properties mit den Verbindungsdaten für H2.

Hinweis

Verwende wieder die gleiche Struktur wie in den vorigen Aufgaben mit jdbc_url, jdbc_user und jdbc_pwd.

Lösung: connection.properties

jdbc_url=jdbc:h2:mem:testdb;INIT=RUNSCRIPT FROM 'schema.sql'
jdbc_user=sa
jdbc_pwd=

Schritt 2: SQL-Datei erstellen

Lege eine Datei schema.sql an. Diese Datei soll eine Tabelle person erstellen.

Hinweis

Für diesen Vergleich genügt eine einfache Tabelle mit id, name und age.

Lösung: schema.sql

DROP TABLE IF EXISTS person;

CREATE TABLE person (
    id INT PRIMARY KEY,
    name VARCHAR(100),
    age INT
);

Schritt 3: Verbindung herstellen

Lade die Properties-Datei und stelle eine Verbindung zur Datenbank her.

Hinweis

Verwende wieder Files.newInputStream(Path.of("connection.properties")) und DriverManager.getConnection(...).

Schritt 4: Viele Datensätze einzeln einfügen

Erstelle eine Methode, die viele Personen mit einzelnen executeUpdate()-Aufrufen einfügt.

Hinweis

Verwende eine Schleife und führe für jeden Datensatz ein eigenes INSERT aus. 

insertStmt.executeUpdate();

Miss die Zeit vor und nach dem Einfügen, zum Beispiel mit: 

long start = System.nanoTime();
long end = System.nanoTime();

Schritt 5: Tabelle wieder leeren

Leere nach dem ersten Durchlauf die Tabelle, damit beide Varianten unter denselben Bedingungen getestet werden.

Hinweis

Verwende dafür zum Beispiel: 

DELETE FROM person

Führe diese Anweisung ebenfalls mit executeUpdate() aus.

Schritt 6: Viele Datensätze mit Batch einfügen

Erstelle nun eine zweite Methode, die dieselben Personen mit addBatch() sammelt und am Ende mit executeBatch() ausführt.

Hinweis

Setze in der Schleife die Werte und füge jede Anweisung zur Batch-Liste hinzu: 

insertStmt.addBatch();

Am Ende führst du alle gesammelten Befehle gemeinsam aus: 

insertStmt.executeBatch();

Miss auch hier wieder die Ausführungszeit.

Schritt 7: Zeiten vergleichen

Gib auf der Konsole aus, wie lange beide Varianten gedauert haben.

Hinweis

Rechne die gemessene Zeit zum Beispiel in Millisekunden um und gib sie aus. 

System.out.println("Ohne Batch: " + duration + " ms");
System.out.println("Mit Batch: " + duration + " ms");

Schritt 8: Ergebnis interpretieren

Vergleiche die beiden Zeitmessungen und überlege, warum Batch Processing oft schneller ist.

Hinweis

Beim Batch Processing werden viele SQL-Anweisungen gesammelt und gemeinsam an die Datenbank geschickt. Dadurch sind oft weniger einzelne Datenbankzugriffe nötig.

Erklärung

Beim Batch Processing werden mehrere SQL-Anweisungen gesammelt und anschließend gemeinsam ausgeführt.

Häufig verwendete Methoden sind:

  • addBatch() → fügt eine Anweisung zur Batch-Liste hinzu
  • executeBatch() → führt alle gesammelten Anweisungen aus

Das ist besonders nützlich, wenn sehr viele Datensätze eingefügt, geändert oder gelöscht werden sollen.

Erklärung

executeUpdate() verwendet man, wenn man eine einzelne SQL-Anweisung ausführen möchte, die Daten verändert.

  • INSERT
  • UPDATE
  • DELETE

executeBatch() verwendet man, wenn mehrere solcher Anweisungen gesammelt und gemeinsam ausgeführt werden sollen.

Lösung

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.PreparedStatement;
import java.sql.SQLException;
import java.io.InputStream;
import java.nio.file.Files;
import java.nio.file.Path;
import java.util.Properties;

public class JdbcBatchExample {

    private static final int PERSON_COUNT = 5000;

    public static void main(String[] args) {

        Properties props = new Properties();
        Path path = Path.of("connection.properties");

        if (!Files.exists(path)) {
            System.out.println("Properties-Datei nicht gefunden!");
            return;
        }

        try (InputStream is = Files.newInputStream(path)) {

            props.load(is);

            String url = props.getProperty("jdbc_url");
            String user = props.getProperty("jdbc_user");
            String pwd = props.getProperty("jdbc_pwd");

            try (Connection conn = DriverManager.getConnection(url, user, pwd)) {

                long normalTime = insertNormally(conn);
                System.out.println("Ohne Batch: " + normalTime + " ms");

                clearTable(conn);

                long batchTime = insertWithBatch(conn);
                System.out.println("Mit Batch: " + batchTime + " ms");

            } catch (SQLException e) {
                System.out.println("Fehler bei der Datenbankoperation:");
                e.printStackTrace();
            }

        } catch (Exception e) {
            System.out.println("Fehler beim Laden der Properties:");
            e.printStackTrace();
        }
    }

    private static long insertNormally(Connection conn) throws SQLException {
        String sql = "INSERT INTO person VALUES (?, ?, ?)";
        long start = System.nanoTime();

        try (PreparedStatement insertStmt = conn.prepareStatement(sql)) {
            for (int i = 1; i <= PERSON_COUNT; i++) {
                insertStmt.setInt(1, i);
                insertStmt.setString(2, "Person_" + i);
                insertStmt.setInt(3, 18 + (i % 50));
                insertStmt.executeUpdate();
            }
        }

        long end = System.nanoTime();
        return (end - start) / 1_000_000;
    }

    private static void clearTable(Connection conn) throws SQLException {
        try (PreparedStatement deleteStmt = conn.prepareStatement("DELETE FROM person")) {
            deleteStmt.executeUpdate();
        }
    }

    private static long insertWithBatch(Connection conn) throws SQLException {
        String sql = "INSERT INTO person VALUES (?, ?, ?)";
        long start = System.nanoTime();

        try (PreparedStatement insertStmt = conn.prepareStatement(sql)) {
            for (int i = 1; i <= PERSON_COUNT; i++) {
                insertStmt.setInt(1, i);
                insertStmt.setString(2, "Person_" + i);
                insertStmt.setInt(3, 18 + (i % 50));
                insertStmt.addBatch();
            }

            insertStmt.executeBatch();
        }

        long end = System.nanoTime();
        return (end - start) / 1_000_000;
    }
}
