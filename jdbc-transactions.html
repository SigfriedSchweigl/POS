Alles oder nichts – commit und rollback

Aufgabe:

Erstelle ein Java-Programm, das zwei Datenbankoperationen ausführt:

  • eine Person einfügen
  • eine andere Person löschen

Beide Operationen sollen Teil einer Transaktion sein.

Wenn ein Fehler auftritt, sollen alle Änderungen rückgängig gemacht werden.

Ziel

In dieser Aufgabe lernst du, wie mehrere SQL-Anweisungen als eine gemeinsame Transaktion ausgeführt werden.

Entweder werden alle Änderungen gespeichert (commit()) oder bei einem Fehler werden alle Änderungen zurückgenommen (rollback()).

Merke: Eine Transaktion folgt dem Prinzip „alles oder nichts“.

Wichtige Methoden

  • setAutoCommit(false) → automatische Speicherung ausschalten
  • commit() → Änderungen dauerhaft speichern
  • rollback() → Änderungen rückgängig machen
Standardmäßig ist oft autoCommit = true. Dann wird jede SQL-Anweisung sofort gespeichert. Für Transaktionen muss man deshalb setAutoCommit(false) setzen.

Benötigte Dateien

1. db.properties

Die Verbindungsdaten werden – wie in den anderen Aufgaben – aus einer Properties-Datei geladen.

url=jdbc:h2:./db/personendb
user=sa
password=

2. schema.sql

Dieses SQL-Skript erzeugt die Tabelle person und fügt ein paar Testdaten ein. 

DROP TABLE IF EXISTS person;

CREATE TABLE person (
    id INT PRIMARY KEY,
    vorname VARCHAR(100) NOT NULL,
    alter INT NOT NULL
);

INSERT INTO person (id, vorname, alter) VALUES (1, 'Anna', 20);
INSERT INTO person (id, vorname, alter) VALUES (2, 'Ben', 22);
INSERT INTO person (id, vorname, alter) VALUES (3, 'Clara', 19);
Anfangszustand der Tabelle:
1 – Anna – 20
2 – Ben – 22
3 – Clara – 19

Java-Beispiel

Das Programm führt zwei Durchläufe aus:

  1. ein erfolgreicher Durchlauf mit commit()
  2. ein fehlerhafter Durchlauf mit rollback()
import java.io.FileInputStream;
import java.io.IOException;
import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.PreparedStatement;
import java.sql.ResultSet;
import java.sql.SQLException;
import java.util.Properties;

public class TransactionDemo {

    public static void main(String[] args) {
        Properties props = new Properties();

        try (FileInputStream fis = new FileInputStream("db.properties")) {
            props.load(fis);
        } catch (IOException e) {
            System.out.println("Fehler beim Laden der db.properties");
            e.printStackTrace();
            return;
        }

        String url = props.getProperty("url");
        String user = props.getProperty("user");
        String password = props.getProperty("password");

        try (Connection con = DriverManager.getConnection(url, user, password)) {

            System.out.println("=== Anfangszustand ===");
            printPersons(con);

            System.out.println("\n=== 1. Durchlauf: commit ===");
            runTransactionWithCommit(con);
            printPersons(con);

            System.out.println("\n=== 2. Durchlauf: rollback ===");
            runTransactionWithRollback(con);
            printPersons(con);

        } catch (SQLException e) {
            System.out.println("Datenbankfehler:");
            e.printStackTrace();
        }
    }

    private static void runTransactionWithCommit(Connection con) throws SQLException {
        String insertSql = "INSERT INTO person (id, vorname, alter) VALUES (?, ?, ?)";
        String deleteSql = "DELETE FROM person WHERE id = ?";

        con.setAutoCommit(false);

        try (
            PreparedStatement insertStmt = con.prepareStatement(insertSql);
            PreparedStatement deleteStmt = con.prepareStatement(deleteSql)
        ) {
            insertStmt.setInt(1, 4);
            insertStmt.setString(2, "David");
            insertStmt.setInt(3, 25);
            insertStmt.executeUpdate();

            deleteStmt.setInt(1, 2);
            deleteStmt.executeUpdate();

            con.commit();
            System.out.println("Transaktion erfolgreich: commit durchgeführt.");

        } catch (SQLException e) {
            con.rollback();
            System.out.println("Fehler: rollback durchgeführt.");
            throw e;
        } finally {
            con.setAutoCommit(true);
        }
    }

    private static void runTransactionWithRollback(Connection con) throws SQLException {
        String insertSql = "INSERT INTO person (id, vorname, alter) VALUES (?, ?, ?)";
        String wrongDeleteSql = "DELETE FROM person WHERE falschesfeld = ?";

        con.setAutoCommit(false);

        try (
            PreparedStatement insertStmt = con.prepareStatement(insertSql);
            PreparedStatement deleteStmt = con.prepareStatement(wrongDeleteSql)
        ) {
            insertStmt.setInt(1, 5);
            insertStmt.setString(2, "Eva");
            insertStmt.setInt(3, 23);
            insertStmt.executeUpdate();

            deleteStmt.setInt(1, 1);
            deleteStmt.executeUpdate();

            con.commit();
            System.out.println("Transaktion erfolgreich: commit durchgeführt.");

        } catch (SQLException e) {
            con.rollback();
            System.out.println("Fehler erkannt: rollback durchgeführt.");
            System.out.println("Grund: " + e.getMessage());
        } finally {
            con.setAutoCommit(true);
        }
    }

    private static void printPersons(Connection con) throws SQLException {
        String sql = "SELECT id, vorname, alter FROM person ORDER BY id";

        try (
            PreparedStatement pstmt = con.prepareStatement(sql);
            ResultSet rs = pstmt.executeQuery()
        ) {
            while (rs.next()) {
                int id = rs.getInt("id");
                String vorname = rs.getString("vorname");
                int alter = rs.getInt("alter");

                System.out.println(id + " | " + vorname + " | " + alter);
            }
        }
    }
}

Was passiert im Programm?

Schritt 1: Anfangszustand anzeigen

Zu Beginn enthält die Tabelle drei Personen:

1 | Anna | 20
2 | Ben | 22
3 | Clara | 19

Schritt 2: Erfolgreiche Transaktion

Es wird eine neue Person eingefügt: David.

Danach wird eine andere Person gelöscht: Ben.

Da beide SQL-Befehle erfolgreich sind, wird am Ende commit() ausgeführt.

1 | Anna | 20
3 | Clara | 19
4 | David | 25

Schritt 3: Fehlerhafte Transaktion

Danach wird wieder eine Person eingefügt: Eva.

Anschließend soll eine andere Person gelöscht werden. Dabei wird aber absichtlich ein Fehler eingebaut: 

DELETE FROM person WHERE falschesfeld = ?

Die Spalte falschesfeld existiert nicht. Deshalb tritt ein SQL-Fehler auf.

In diesem Fall wird rollback() ausgeführt. Dadurch wird auch das vorherige Einfügen von Eva wieder rückgängig gemacht.

1 | Anna | 20
3 | Clara | 19
4 | David | 25
Obwohl im zweiten Durchlauf die neue Person zuerst erfolgreich eingefügt wurde, bleibt sie am Ende nicht in der Tabelle. Der Grund ist das rollback().

Erwartete Konsolenausgabe

=== Anfangszustand ===
1 | Anna | 20
2 | Ben | 22
3 | Clara | 19

=== 1. Durchlauf: commit ===
Transaktion erfolgreich: commit durchgeführt.
1 | Anna | 20
3 | Clara | 19
4 | David | 25

=== 2. Durchlauf: rollback ===
Fehler erkannt: rollback durchgeführt.
1 | Anna | 20
3 | Clara | 19
4 | David | 25

Arbeitsaufträge

  1. Warum gehören Einfügen und Löschen hier zu einer gemeinsamen Transaktion?
  2. Warum wird setAutoCommit(false) verwendet?
  3. Was bewirkt commit()?
  4. Was bewirkt rollback()?
  5. Warum erscheint Eva nach dem zweiten Durchlauf nicht in der Tabelle?

Bonus

Erweitere das Programm:

  • Gib vor und nach jeder Transaktion die Anzahl der Personen aus.
  • Baue einen eigenen Fehlertext für den Rollback-Fall ein.
  • Ändere das Beispiel so, dass statt Löschen ein UPDATE durchgeführt wird.

Merksätze

  • Eine Transaktion fasst mehrere SQL-Anweisungen zu einer Einheit zusammen.
  • Mit commit() werden Änderungen dauerhaft gespeichert.
  • Mit rollback() werden Änderungen der Transaktion rückgängig gemacht.
  • Bei Transaktionen gilt: alles oder nichts.
