Ziel

Ziel ist es, mehrere Datenbankoperationen als Transaktion zusammenzufassen. Dabei soll gezeigt werden, dass entweder alle Änderungen gespeichert werden oder bei einem Fehler alle Änderungen rückgängig gemacht werden.

Aufgabe 7 – Transaktionen

Alles oder nichts – commit und rollback

Aufgabe:

Erstelle ein Java-Programm, das zwei Datenbankoperationen ausführt:

  • eine Person einfügen
  • eine andere Person löschen

Beide Operationen sollen Teil einer Transaktion sein.

Wenn ein Fehler auftritt, sollen alle Änderungen rückgängig gemacht werden.

Das Programm soll zwei Fälle zeigen:

  • einmal commit → beide Operationen werden erfolgreich gespeichert
  • einmal rollback → bei einem Fehler werden alle Änderungen zurückgenommen

Gib nach jedem Durchlauf den Inhalt der Tabelle auf der Konsole aus.

Schritt 1: Properties-Datei verwenden

Verwende wieder eine Datei connection.properties mit den Verbindungsdaten für H2.

Hinweis

Du kannst dieselbe Properties-Datei wie in den vorigen Aufgaben verwenden.

Lösung: connection.properties

jdbc_url=jdbc:h2:mem:testdb;INIT=RUNSCRIPT FROM 'schema.sql'
jdbc_user=sa
jdbc_pwd=

Schritt 2: SQL-Datei erstellen

Lege eine Datei schema.sql an. Diese Datei soll eine Tabelle person erstellen und einige Beispieldaten einfügen.

Hinweis

Verwende wieder die Spalten id, name und age.

Lösung: schema.sql

DROP TABLE IF EXISTS person;

CREATE TABLE person (
    id INT PRIMARY KEY,
    name VARCHAR(100),
    age INT
);

INSERT INTO person VALUES (1, 'Anna', 22);
INSERT INTO person VALUES (2, 'Max', 25);
INSERT INTO person VALUES (3, 'Lisa', 19);

Schritt 3: Verbindung herstellen

Lade die Properties-Datei und stelle eine Verbindung zur Datenbank her.

Hinweis

Verwende wieder Files.newInputStream(Path.of("connection.properties")) und DriverManager.getConnection(...).

Schritt 4: Auto-Commit deaktivieren

Deaktiviere das automatische Speichern der einzelnen SQL-Anweisungen.

Hinweis

Standardmäßig ist bei JDBC autoCommit = true. Das bedeutet: Jede SQL-Anweisung wird sofort dauerhaft in der Datenbank gespeichert.

Für Transaktionen musst du daher setAutoCommit(false) verwenden. 

conn.setAutoCommit(false);

Schritt 5: Erste Transaktion erfolgreich durchführen

Führe nun zwei zusammengehörige SQL-Anweisungen aus:

  • eine neue Person einfügen
  • eine bestehende Person löschen

Beide Anweisungen sollen innerhalb einer Transaktion ausgeführt werden.

Verwende dafür executeUpdate(), da Daten verändert werden.

Wenn beide Befehle erfolgreich sind, sollen die Änderungen mit commit() dauerhaft gespeichert werden.

Hinweis

Verwende zwei SQL-Anweisungen: 

INSERT INTO person VALUES (?, ?, ?)
DELETE FROM person WHERE id = ?

Führe die Anweisungen mit executeUpdate() aus: 

insertStmt.executeUpdate();
  deleteStmt.executeUpdate();

executeUpdate() wird verwendet für:

  • INSERT
  • UPDATE
  • DELETE

Wenn beide Befehle erfolgreich waren, rufe am Ende commit() auf: 

conn.commit();

Dadurch werden beide Änderungen dauerhaft in der Datenbank gespeichert.

Schritt 6: Zweite Transaktion mit Fehler erzeugen

Führe erneut ein Einfügen und ein Löschen aus, baue diesmal aber absichtlich einen Fehler in die zweite SQL-Anweisung ein.

Hinweis

Du kannst zum Beispiel eine Spalte verwenden, die es nicht gibt: 

DELETE FROM person WHERE falschesfeld = ?

Dadurch entsteht ein SQL-Fehler.

Schritt 7: Rollback ausführen

Wenn ein Fehler auftritt, sollen alle Änderungen der laufenden Transaktion zurückgenommen werden.

Hinweis

Fange die Exception ab und rufe dann rollback() auf. 

catch (SQLException e) {
    conn.rollback();
}

Schritt 8: Tabelleninhalt ausgeben

Gib vor der ersten Transaktion, nach dem commit und nach dem rollback den Tabelleninhalt aus.

Hinweis

Verwende eine Methode, die alle Personen mit einer SELECT-Abfrage ausliest und auf der Konsole ausgibt. 

SELECT * FROM person ORDER BY id

Erklärung

Eine Transaktion fasst mehrere Datenbankänderungen zu einer Einheit zusammen.

Das bedeutet:

  • entweder werden alle Änderungen übernommen
  • oder keine einzige Änderung bleibt gespeichert

Genau dafür gibt es:

  • commit() → speichert alle Änderungen dauerhaft
  • rollback() → macht alle Änderungen der Transaktion rückgängig

Lösung

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.PreparedStatement;
import java.sql.ResultSet;
import java.sql.SQLException;
import java.io.InputStream;
import java.nio.file.Files;
import java.nio.file.Path;
import java.util.Properties;

public class JdbcTransactionExample {

    public static void main(String[] args) {

        Properties props = new Properties();
        Path path = Path.of("connection.properties");

        if (!Files.exists(path)) {
            System.out.println("Properties-Datei nicht gefunden!");
            return;
        }

        try (InputStream is = Files.newInputStream(path)) {

            props.load(is);

            String url = props.getProperty("jdbc_url");
            String user = props.getProperty("jdbc_user");
            String pwd = props.getProperty("jdbc_pwd");

            try (Connection conn = DriverManager.getConnection(url, user, pwd)) {

                System.out.println("Anfangszustand:");
                printPersons(conn);

                runCommitExample(conn);

                System.out.println("Nach commit:");
                printPersons(conn);

                runRollbackExample(conn);

                System.out.println("Nach rollback:");
                printPersons(conn);

            } catch (SQLException e) {
                System.out.println("Fehler bei der Datenbankoperation:");
                e.printStackTrace();
            }

        } catch (Exception e) {
            System.out.println("Fehler beim Laden der Properties:");
            e.printStackTrace();
        }
    }

    private static void runCommitExample(Connection conn) throws SQLException {
        String insertSql = "INSERT INTO person VALUES (?, ?, ?)";
        String deleteSql = "DELETE FROM person WHERE id = ?";

        conn.setAutoCommit(false);

        try (PreparedStatement insertStmt = conn.prepareStatement(insertSql);
             PreparedStatement deleteStmt = conn.prepareStatement(deleteSql)) {

            insertStmt.setInt(1, 4);
            insertStmt.setString(2, "Tom");
            insertStmt.setInt(3, 30);
            insertStmt.executeUpdate();

            deleteStmt.setInt(1, 2);
            deleteStmt.executeUpdate();

            conn.commit();
            System.out.println("commit wurde durchgeführt.");
            System.out.println();

        } catch (SQLException e) {
            conn.rollback();
            System.out.println("Rollback wegen Fehler im commit-Beispiel.");
            System.out.println();
        } finally {
            conn.setAutoCommit(true);
        }
    }

    private static void runRollbackExample(Connection conn) throws SQLException {
        String insertSql = "INSERT INTO person VALUES (?, ?, ?)";
        String wrongDeleteSql = "DELETE FROM person WHERE falschesfeld = ?";

        conn.setAutoCommit(false);

        try (PreparedStatement insertStmt = conn.prepareStatement(insertSql);
             PreparedStatement deleteStmt = conn.prepareStatement(wrongDeleteSql)) {

            insertStmt.setInt(1, 5);
            insertStmt.setString(2, "Eva");
            insertStmt.setInt(3, 27);
            insertStmt.executeUpdate();

            deleteStmt.setInt(1, 1);
            deleteStmt.executeUpdate();

            conn.commit();
            System.out.println("commit wurde durchgeführt.");
            System.out.println();

        } catch (SQLException e) {
            conn.rollback();
            System.out.println("rollback wurde durchgeführt.");
            System.out.println();
        } finally {
            conn.setAutoCommit(true);
        }
    }

    private static void printPersons(Connection conn) throws SQLException {
        String sql = "SELECT * FROM person ORDER BY id";

        try (PreparedStatement pstmt = conn.prepareStatement(sql);
             ResultSet rs = pstmt.executeQuery()) {

            while (rs.next()) {
                int id = rs.getInt("id");
                String name = rs.getString("name");
                int age = rs.getInt("age");

                System.out.println(id + " | " + name + " | " + age);
            }

            System.out.println();
        }
    }
}
