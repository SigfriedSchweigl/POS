JDBC – Aufgabe 1: Datenbankverbindung testen

Verbindung zu H2 und SQL Server mit mehreren Schritten und Hinweisen

Start JDBC H2 SQL Server

Ziel der Aufgabe

In dieser Aufgabe soll eine erste JDBC-Verbindung aufgebaut und getestet werden. Es gibt zwei Varianten:

  • H2 als einfache lokale Testdatenbank
  • SQL Server als Beispiel für eine externe Datenbank

Arbeite die Schritte nacheinander durch und versuche zuerst selbst eine Lösung zu entwickeln. Die Hinweise helfen dir dabei, ohne sofort die komplette Lösung anzusehen.

Aufgabe 1A – H2 Connection Test

Verbindung zu einer H2-Datenbank herstellen

Aufgabe:

Schreibe ein Java-Programm, das eine Verbindung zu einer H2-Datenbank aufbaut und anschließend ausgibt, ob die Verbindung erfolgreich war.

Schritt 1

Lege ein Java-Projekt an und binde den H2-Treiber ein.

Schritt 2

Verwende folgende Verbindungsdaten:

URL: jdbc:h2:~/test
User: sa
Passwort:

Schritt 3

Verwende DriverManager.getConnection(...), um die Verbindung zu öffnen.

Schritt 4

Prüfe, ob das zurückgegebene Connection-Objekt nicht null ist, und gib eine passende Meldung aus.

Schritt 5

Schließe die Verbindung sauber wieder.

Hinweis 1

Du brauchst meist diese Imports: 

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.SQLException;

Hinweis 2

Die zentrale Methode sieht so aus: 

Connection connection = DriverManager.getConnection(url, user, password);

Hinweis 3

Nutze am besten try-with-resources, damit die Verbindung automatisch geschlossen wird.

Mögliche Lösung

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.SQLException;

public class H2ConnectionTest {
    public static void main(String[] args) {
        String url = "jdbc:h2:~/test";
        String user = "sa";
        String password = "";

        try (Connection connection = DriverManager.getConnection(url, user, password)) {
            if (connection != null) {
                System.out.println("Verbindung zur H2-Datenbank erfolgreich hergestellt.");
            }
        } catch (SQLException e) {
            System.out.println("Fehler beim Verbinden mit H2.");
            e.printStackTrace();
        }
    }
}

Diese Lösung verwendet try-with-resources. Dadurch wird die Verbindung nach der Benutzung automatisch geschlossen.

Aufgabe 1B – SQL Server Connection Test

Verbindung zu einem SQL Server herstellen

Aufgabe:

Schreibe ein Java-Programm, das eine Verbindung zu einem SQL Server herstellt und danach eine Erfolgsmeldung ausgibt.

Schritt 1

Binde den JDBC-Treiber für SQL Server in dein Projekt ein.

Schritt 2

Verwende folgende Beispiel-Verbindungsdaten:

URL: jdbc:sqlserver://localhost:1433;databaseName=School;encrypt=true;trustServerCertificate=true
User: sa
Passwort: DeinPasswort

Schritt 3

Baue mit DriverManager.getConnection(...) die Verbindung auf.

Schritt 4

Gib bei Erfolg eine Meldung aus, etwa: Verbindung erfolgreich hergestellt.

Schritt 5

Fange mögliche Fehler mit SQLException ab.

Hinweis 1

Der Unterschied zu H2 liegt vor allem in der URL und im verwendeten Treiber.

Hinweis 2

Achte darauf, dass der SQL Server läuft und dass Port, Datenbankname, Benutzername und Passwort korrekt sind.

Hinweis 3

Falls Zertifikatsfehler auftreten, hilft in Testumgebungen oft: trustServerCertificate=true

Mögliche Lösung

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.SQLException;

public class SqlServerConnectionTest {
    public static void main(String[] args) {
        String url = "jdbc:sqlserver://localhost:1433;databaseName=School;encrypt=true;trustServerCertificate=true";
        String user = "sa";
        String password = "DeinPasswort";

        try (Connection connection = DriverManager.getConnection(url, user, password)) {
            if (connection != null) {
                System.out.println("Verbindung zum SQL Server erfolgreich hergestellt.");
            }
        } catch (SQLException e) {
            System.out.println("Fehler beim Verbinden mit dem SQL Server.");
            e.printStackTrace();
        }
    }
}

In realen Projekten sollten Zugangsdaten nicht direkt im Code stehen, sondern z. B. über eine properties-Datei geladen werden.
