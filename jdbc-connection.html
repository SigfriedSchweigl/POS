JDBC – Aufgabe 1: Datenbankverbindung erstellen

Schrittweise Verbindung zu H2 und SQL Server mit Hinweisen

Start JDBC H2 SQL Server

Ziel

Ziel ist es, eine Verbindung zu einer Datenbank herzustellen und mit DatabaseMetaData Informationen über die Datenbank und den JDBC-Treiber auszulesen.

Aufgabe 1A – H2

Connection zur H2-Datenbank

Aufgabe:

Stelle eine Verbindung zu einer H2-Datenbank her. Lies anschließend die Metadaten der Verbindung aus und gib mindestens folgende Informationen auf der Konsole aus:

  • Name des JDBC-Treibers
  • Version des JDBC-Treibers
  • Name des Datenbankprodukts
  • Version des Datenbankprodukts

Schritt 1: Maven Dependency hinzufügen

Füge den H2-Treiber in deine pom.xml ein.

Hinweis

Dependencies findest du auf der offiziellen Maven-Seite:

Maven Repository 

<dependencies>
  ...
  <dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <version>2.4.240</version>
  </dependency>
</dependencies>

Schritt 2: Verbindungsdaten festlegen

Verwende folgende Daten für die Verbindung:

URL: jdbc:h2:mem:testdb
User: sa
Passwort:

Hinweis

Die URL beginnt bei H2 immer mit jdbc:h2:. Die Verbindungsdaten können hardcoded als String gespeichert werden, keine Properties wie in den Folien beispielhaft dargestellt, nötig.

Erklärung

mem bedeutet, dass H2 eine In-Memory-Datenbank verwendet.

Die Datenbank liegt nur im Arbeitsspeicher und benötigt daher keine Datei auf der Festplatte.

Sie existiert nur, solange das Programm läuft.

Schritt 3: Verbindung herstellen

Nutze JDBC, um eine Verbindung zur Datenbank aufzubauen.

Hinweis

Verwende DriverManager.getConnection(...).

Schritt 4: Datenbankinformationen ausgeben

Lies die Metadaten der Verbindung aus und gib mindestens folgende Informationen aus:

  • Treibername
  • Treiberversion
  • Datenbankname
  • Datenbankversion

Hinweis

Verwende connection.getMetaData(), um ein DatabaseMetaData-Objekt zu erhalten.

Mit Methoden wie getDriverName(), getDriverVersion(), getDatabaseProductName() und getDatabaseProductVersion() kannst du die Informationen auslesen.

Lösung

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.DatabaseMetaData;
import java.sql.SQLException;

public class H2Test {
    public static void main(String[] args) {

        String url = "jdbc:h2:mem:testdb";
        String user = "sa";
        String password = "";

        try (Connection connection = DriverManager.getConnection(url, user, password)) {

            System.out.println("Verbunden mit Datenbank...");

            DatabaseMetaData dbmd = connection.getMetaData();

            System.out.println("Treibername = " + dbmd.getDriverName());
            System.out.println("Treiberversion = " + dbmd.getDriverVersion());
            System.out.println("Datenbankname = " + dbmd.getDatabaseProductName());
            System.out.println("Datenbankversion = " + dbmd.getDatabaseProductVersion());

        } catch (SQLException e) {
            e.printStackTrace();
        }
    }
}
Aufgabe 1B – SQL Server

Connection zu SQL Server

Aufgabe:

Stelle eine Verbindung zu einem SQL Server her. Lies anschließend die Metadaten der Verbindung aus und gib mindestens folgende Informationen auf der Konsole aus:

  • Name des JDBC-Treibers
  • Version des JDBC-Treibers
  • Name des Datenbankprodukts
  • Version des Datenbankprodukts

Schritt 1: Maven Dependency hinzufügen

Füge den JDBC-Treiber für SQL Server in deine pom.xml ein.

Hinweis

Suche auf mvnrepository.com nach: mssql-jdbc

Schritt 2: Verbindungsdaten festlegen

Verwende eine JDBC-URL nach folgendem Muster:

jdbc:sqlserver://localhost:1433;databaseName=DBNAME;encrypt=true;trustServerCertificate=true

Hinweis

Achte darauf, dass dein SQL Server läuft und die Portnummer richtig ist. Ersetze DBNAME durch den Namen deiner Datenbank.

Schritt 3: Verbindung herstellen

Nutze JDBC, um eine Verbindung zum SQL Server aufzubauen.

Hinweis

Die Vorgehensweise ist gleich wie bei H2: DriverManager.getConnection(...)

Schritt 4: Datenbankinformationen ausgeben

Lies die Metadaten der Verbindung aus und gib mindestens folgende Informationen aus:

  • Treibername
  • Treiberversion
  • Datenbankname
  • Datenbankversion

Hinweis

Verwende auch hier connection.getMetaData(), um ein DatabaseMetaData-Objekt zu erhalten.

Die benötigten Informationen kannst du mit denselben Methoden wie bei H2 auslesen.

Lösung

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.DatabaseMetaData;
import java.sql.SQLException;

public class SqlTest {
    public static void main(String[] args) {

        String url = "jdbc:sqlserver://localhost:1433;databaseName=DBNAME;encrypt=true;trustServerCertificate=true";
        String user = "sa";
        String password = "";

        try (Connection connection = DriverManager.getConnection(url, user, password)) {

            System.out.println("Verbunden mit Datenbank...");

            DatabaseMetaData dbmd = connection.getMetaData();

            System.out.println("Treibername = " + dbmd.getDriverName());
            System.out.println("Treiberversion = " + dbmd.getDriverVersion());
            System.out.println("Datenbankname = " + dbmd.getDatabaseProductName());
            System.out.println("Datenbankversion = " + dbmd.getDatabaseProductVersion());

        } catch (SQLException e) {
            e.printStackTrace();
        }
    }
}
