JDBC – Aufgabe 1

Repository für Haustiere mit H2-Datenbank umsetzen

Start JDBC Aufgabe 1

Ziel

In dieser Aufgabe setzt du ein kleines JDBC-Beispiel mit einem Repository um. Dabei arbeitest du mit einem Record, einem Enum, einem Repository-Interface und einer H2-In-Memory-Datenbank.

Ziel ist es, die typischen CRUD-Operationen mit JDBC praktisch anzuwenden: speichern, laden, aktualisieren und löschen.

Aufgabenstellung

Erstelle eine kleine Anwendung zur Verwaltung von Haustieren mit JDBC.

  • Erstelle ein Record Pet mit den Attributen: id, name, type und birthdate.
  • Erstelle ein Enum PetType mit den Werten: DOG, CAT und HAMSTER.
  • Erstelle ein Interface PetRepository mit folgenden Methoden: 
      Pet save(Pet pet) throws SQLException;
  List<Pet> findAll() throws SQLException;
  Optional<Pet> findById(long id) throws SQLException;
  void update(Pet pet) throws SQLException;
  void deleteById(long id) throws SQLException;
  • Implementiere das Interface in einer Klasse JdbcPetRepository.
  • Erstelle eine main-Methode, die:
    • eine H2-In-Memory-Datenbank verwendet
    • die Tabelle pet anlegt
    • einige Testdaten einfügt
  • Teste alle Methoden des JdbcPetRepository.

Hinweis

Überlege dir zuerst die Reihenfolge: zuerst Enum und Record, dann Interface, danach Repository-Klasse und am Ende die main-Methode mit Datenbankaufbau und Tests.

Für birthdate passt in Java gut LocalDate. In der Datenbank kannst du dafür den Typ DATE verwenden.

public enum PetType {
    DOG,
    CAT,
    HAMSTER
}
    
import java.time.LocalDate;

public record Pet(long id, String name, PetType type, LocalDate birthdate) {
}

Beispiel für das Interface

import java.sql.SQLException;
import java.util.List;
import java.util.Optional;

public interface PetRepository {
    Pet save(Pet pet) throws SQLException;
    List<Pet> findAll() throws SQLException;
    Optional<Pet> findById(long id) throws SQLException;
    void update(Pet pet) throws SQLException;
    void deleteById(long id) throws SQLException;
}

Beispiel für die main-Methode

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.SQLException;
import java.sql.Statement;
import java.time.LocalDate;

public class App {
    public static void main(String[] args) throws SQLException {
        String url = "jdbc:h2:mem:petdb";

        try (Connection connection = DriverManager.getConnection(url);
             Statement stmt = connection.createStatement()) {

            stmt.execute("""
                CREATE TABLE pet (
                    id IDENTITY,
                    name VARCHAR(100) NOT NULL,
                    type VARCHAR(20) NOT NULL,
                    birthdate DATE NOT NULL
                )
            """);

            JdbcPetRepository repo = new JdbcPetRepository(connection);

            Pet luna = repo.save(new Pet(null, "Luna", PetType.CAT , LocalDate.of(2019, 5, 10)));
            Pet buddy = repo.save(new Pet(null, "Buddy", PetType.DOG, LocalDate.of(2018, 3, 22)));

            System.out.println("\n--- All pets ---");
            repo.findAll().forEach(System.out::println);

            System.out.println("\n--- Find Luna by ID ---");
            repo.findById(luna.id()).ifPresent(System.out::println);

            System.out.println("\n--- Update Buddy from DOG to HAMSTER ---");
            Pet updatedBuddy = new Pet(buddy.id(), "Buddy", PetType.HAMSTER, buddy.birthdate());
            repo.update(updatedBuddy);
            System.out.println(repo.findById(buddy.id()).orElseThrow());

            System.out.println("\n--- Delete Luna ---");
            repo.deleteById(luna.id());
            repo.findAll().forEach(System.out::println);
        }
    }
}

Was getestet werden soll

- save(...) fügt ein neues Haustier ein
- findAll() liefert alle Haustiere
- findById(...) liefert ein Optional<Pet>
- update(...) ändert vorhandene Daten
- deleteById(...) löscht einen Datensatz
