JDBC – Debugging

Finde und behebe typische Fehler

Start Ziel Aufgabe

Ziel

In dieser Aufgabe analysierst du fehlerhaften JDBC-Code. Ziel ist es, typische Fehler zu erkennen und zu verstehen.

Solche Fehler passieren in der Praxis sehr oft – deshalb ist Debugging eine wichtige Fähigkeit.

Fehleranalyse: Warum funktioniert der Code nicht?

Der folgende Code soll alle Schauspieler aus der Datenbank laden. Er enthält jedoch mehrere Fehler. 

Connection con = DriverManager.getConnection(url, user, password);

Statement stmt = con.createStatement();

ResultSet rs = stmt.executeQuery("SELECT * FROM actor");

while (rs.next()) {
    int id = rs.getInt("actor_id");
    String firstName = rs.getString("first_name");
    String lastName = rs.getString("last_name");

    System.out.println(id + " " + firstName + " " + lastName);
}

rs.close();
stmt.close();

// Connection wird NICHT geschlossen

Schritt 1: Was fehlt im Code?

Schau dir an, welche Ressourcen geöffnet wurden.

Die Connection wird nicht geschlossen.

Das führt zu Ressourcenproblemen (Connection-Leaks).

Schritt 2: Was passiert bei einer Exception?

Was passiert, wenn vorher ein Fehler auftritt?

Wenn eine Exception auftritt, werden close()-Methoden nicht ausgeführt.

→ Ressourcen bleiben offen

Schritt 3: Wie löst man das korrekt?

Es gibt eine moderne Lösung in Java.

Verwende try-with-resources.

Schritt 4: Korrigierter Code

try (Connection con = DriverManager.getConnection(url, user, password);
     Statement stmt = con.createStatement();
     ResultSet rs = stmt.executeQuery("SELECT * FROM actor")) {

    while (rs.next()) {
        int id = rs.getInt("actor_id");
        String firstName = rs.getString("first_name");
        String lastName = rs.getString("last_name");

        System.out.println(id + " " + firstName + " " + lastName);
    }
}

Schritt 5: Bonus – weiterer typischer Fehler

Denke an SQL Injection.

Wenn man Parameter einbaut, sollte man PreparedStatement verwenden, sonst entsteht SQL Injection.
