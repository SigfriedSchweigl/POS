JDBC – Debugging

Finde und behebe typische Fehler

Start JDBC Übung

Ziel

In dieser Übung analysierst du fehlerhaften JDBC-Code. Ziel ist es, typische Fehler zu erkennen und zu verstehen.

Fehleranalyse 1: Warum funktioniert der Code nicht?

Der folgende Code soll einen Schauspieler aus der Datenbank laden. Er enthält jedoch mehrere Fehler. 

Connection con = DriverManager.getConnection(url, user, password);

Statement stmt = con.createStatement();

ResultSet rs = stmt.executeQuery("SELECT * FROM actor WHERE actor_id = 1");

while (rs.next()) {
    int id = rs.getInt("actor_id");
    String firstName = rs.getString("first_name");
    String lastName = rs.getString("last_name");

    System.out.println(id + " " + firstName + " " + lastName);
}

rs.close();
stmt.close();

Schritt 1: Was fehlt im Code?

Schau dir an, welche Ressourcen geöffnet wurden.

Die Connection wird nicht geschlossen.

Das führt zu Ressourcenproblemen.

Schritt 2: Was passiert bei einer Exception?

Was passiert, wenn vorher ein Fehler auftritt?

Wenn eine Exception auftritt, werden close()-Methoden nicht ausgeführt.

→ Ressourcen bleiben offen

Schritt 3: Wie löst man das korrekt?

Verwende try-with-resources.

Schritt 4: Korrigierter Code

try (Connection con = DriverManager.getConnection(url, user, password);
     Statement stmt = con.createStatement();
     ResultSet rs = stmt.executeQuery("SELECT * FROM actor WHERE actor_id = 1")) {

    while (rs.next()) {
        int id = rs.getInt("actor_id");
        String firstName = rs.getString("first_name");
        String lastName = rs.getString("last_name");

        System.out.println(id + " " + firstName + " " + lastName);
    }
}

Schritt 5: Weiterer typischer Fehler

Denke an SQL Injection.

Wenn man Parameter einbaut, sollte man PreparedStatement verwenden.

Fehleranalyse 2: Updateable ResultSet

Der folgende Code soll den Vornamen eines Schauspielers über ein updateable ResultSet ändern. Der Code läuft aber nicht wie erwartet. 

try (Connection con = DriverManager.getConnection(url, user, password);
     Statement stmt = con.createStatement(
         ResultSet.TYPE_SCROLL_INSENSITIVE,
         ResultSet.CONCUR_UPDATABLE
     );
     ResultSet rs = stmt.executeQuery("SELECT actor_id, first_name, last_name FROM actor WHERE actor_id = 1")) {

    if (rs.next()) {
        rs.updateString("first_name", "Max");
    }
}

Schritt 6: Warum wird der neue Wert nicht gespeichert?

Es reicht nicht, nur einen neuen Wert im ResultSet zu setzen.

Der Aufruf updateRow() fehlt.

Mit updateString(...) wird der neue Wert nur im aktuellen Datensatz des ResultSet gesetzt. Erst updateRow() schreibt die Änderung wirklich in die Datenbank.

Schritt 7: Welche Voraussetzung muss zusätzlich erfüllt sein?

Schau dir an, wie Statement und ResultSet erzeugt wurden.

Das ResultSet muss updatable sein.

Deshalb wurde das Statement mit ResultSet.CONCUR_UPDATABLE erstellt.

Schritt 8: Korrigierter Code

try (Connection con = DriverManager.getConnection(url, user, password);
     Statement stmt = con.createStatement(
         ResultSet.TYPE_SCROLL_INSENSITIVE,
         ResultSet.CONCUR_UPDATABLE
     );
     ResultSet rs = stmt.executeQuery("SELECT actor_id, first_name, last_name FROM actor WHERE actor_id = 1")) {

    if (rs.next()) {
        rs.updateString("first_name", "Max");
        rs.updateRow();
    }
}

Schritt 9: Was wäre ein weiterer typischer Fehler?

Überlege, was passiert, wenn man die falsche Art von ResultSet verwendet.

Ein häufiger Fehler ist, dass das Statement nur ein normales read-only ResultSet liefert.

Dann sind Methoden wie updateString(...) oder updateRow() nicht sinnvoll nutzbar.

Schritt 10: Merksatz

Bei einem updateable ResultSet gilt: 

Wert ändern → updateXxx(...) aufrufen
Änderung speichern → updateRow() aufrufen
