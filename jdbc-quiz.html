Abschluss-Quiz

JDBC – Gesamtüberblick

Aufgabe:

Beantworte die folgenden Fragen zu JDBC, ResultSets, Transaktionen, Batch Processing und Architektur.

Frage 1: executeQuery vs. executeUpdate

Wann verwendet man executeQuery() und wann executeUpdate()?

executeQuery() → für SELECT (liefert ResultSet)
executeUpdate() → für INSERT, UPDATE, DELETE (liefert Anzahl der betroffenen Zeilen)

Frage 2: ResultSet Navigation

Welche Methode verwendest du, um direkt zum 3. Datensatz zu springen?

rs.absolute(3)

Frage 3: Code verstehen

ResultSet rs = stmt.executeQuery("SELECT * FROM person");

while (rs.next()) {
    System.out.println(rs.getString("name"));
}

Was macht dieser Code?

Er liest alle Datensätze aus der Tabelle person und gibt den Namen jeder Person aus.

Frage 4: PreparedStatement

Warum sollte man PreparedStatement statt Statement verwenden?

  • Schutz vor SQL Injection
  • bessere Performance
  • klarere Struktur (Platzhalter)

Frage 5: Codefehler erkennen

String sql = "INSERT INTO person VALUES (?, ?, ?)";
PreparedStatement stmt = conn.prepareStatement(sql);

stmt.setInt(1, 1);
stmt.setString(2, "Anna");

stmt.executeUpdate();

Was ist hier das Problem?

Es fehlen Parameter → nicht alle Platzhalter wurden gesetzt.

Frage 6: Transaktionen

Was bewirkt setAutoCommit(false)?

SQL-Anweisungen werden nicht mehr automatisch gespeichert, sondern müssen mit commit() bestätigt werden.

Frage 7: commit vs. rollback

Was ist der Unterschied zwischen commit() und rollback()?

commit() → speichert Änderungen dauerhaft
rollback() → macht Änderungen rückgängig

Frage 8: Code verstehen

conn.setAutoCommit(false);

stmt1.executeUpdate();
stmt2.executeUpdate();

conn.commit();

Warum gehören diese Befehle zusammen?

Beide SQL-Anweisungen bilden eine Transaktion und werden gemeinsam gespeichert.

Frage 9: Batch Processing

Was ist der Vorteil von executeBatch()?

Mehrere SQL-Anweisungen werden gesammelt und gemeinsam ausgeführt → bessere Performance.

Frage 10: Code verstehen

for (int i = 1; i <= 1000; i++) {
    stmt.setInt(1, i);
    stmt.addBatch();
}

stmt.executeBatch();

Was passiert hier?

1000 INSERT-Anweisungen werden gesammelt und anschließend gemeinsam ausgeführt.

Frage 11: Architektur

Welche Aufgabe hat ein Repository-Interface?

Es definiert, welche Methoden verfügbar sind, ohne die Umsetzung festzulegen.

Frage 12: Architektur Vorteil

Warum ist folgende Struktur sinnvoll?

  • domain
  • persistence
  • app

Klare Trennung der Verantwortlichkeiten → besser wartbar und erweiterbar.

Frage 13: Generics

Warum ist Optional<Actor> besser als Optional?

Typsicherheit → kein Casting nötig und weniger Fehler.

Frage 14: Fehler erkennen

Optional actor = repo.findById(1);
System.out.println(actor.get());

Was ist hier problematisch?

Kein Generics + get() kann Exception werfen, wenn leer.

Frage 15: Gesamtverständnis

Warum sollte man JDBC-Code nicht direkt in der Main-Klasse schreiben?

Schlechte Struktur → besser Repository verwenden (Trennung von Logik und DB-Zugriff).
