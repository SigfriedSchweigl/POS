Aufgabe:

Beantworte die folgenden Fragen zu den bisherigen JDBC-Themen. Einige Fragen beziehen sich auf kurze Codefragmente.

Frage 1: Verbindung herstellen

Welche Klasse verwendet man in JDBC typischerweise, um eine Verbindung zur Datenbank herzustellen?

Lösung

Dafür verwendet man typischerweise DriverManager, zum Beispiel: 

Connection conn = DriverManager.getConnection(url, user, pwd);

Frage 2: Properties-Datei

Welche drei Properties wurden in den bisherigen Aufgaben für die Verbindung zur H2-Datenbank verwendet?

Lösung

Verwendet wurden:

  • jdbc_url
  • jdbc_user
  • jdbc_pwd

Frage 3: executeQuery und executeUpdate

Wofür verwendet man executeQuery() und wofür executeUpdate()?

Lösung

executeQuery() verwendet man für SELECT-Abfragen. Die Methode liefert ein ResultSet.

executeUpdate() verwendet man für INSERT, UPDATE und DELETE. Die Methode liefert die Anzahl der geänderten Zeilen.

Frage 4: Code verstehen

ResultSet rs = stmt.executeQuery("SELECT * FROM person");

while (rs.next()) {
    System.out.println(rs.getString("name"));
}

Was macht dieser Code?

Lösung

Der Code führt eine SELECT-Abfrage auf der Tabelle person aus und gibt den Wert der Spalte name für jeden Datensatz aus.

Frage 5: Statement oder PreparedStatement?

Warum verwendet man in der Praxis meist PreparedStatement statt Statement?

Lösung

  • besserer Schutz vor SQL Injection
  • übersichtlichere SQL-Befehle durch Platzhalter ?
  • oft bessere Performance bei wiederholten Anweisungen

Frage 6: Codefehler erkennen

String sql = "INSERT INTO person VALUES (?, ?, ?)";
PreparedStatement stmt = conn.prepareStatement(sql);

stmt.setInt(1, 1);
stmt.setString(2, "Anna");

stmt.executeUpdate();

Was ist hier das Problem?

Lösung

Es wurden nicht alle Platzhalter gesetzt. Der dritte Parameter fehlt.

Frage 7: Generated Keys

Wozu verwendet man Generated Keys?

Lösung

Generated Keys verwendet man, um nach einem INSERT automatisch erzeugte Schlüsselwerte auszulesen, zum Beispiel eine automatisch vergebene ID.

Frage 8: Scrollable ResultSet

Welche Methode verwendest du, um direkt zum zweiten Datensatz zu springen?

Lösung

Dafür verwendet man: 

rs.absolute(2);

Frage 9: Weitere ResultSet-Methoden

Nenne drei Methoden, mit denen man sich in einem scrollbaren ResultSet bewegen kann.

Lösung

Zum Beispiel:

  • first()
  • last()
  • previous()
  • absolute(n)
  • next()

Frage 10: Updatable ResultSet

Welche Methode muss nach updateInt(...) oder updateString(...) aufgerufen werden, damit die Änderung gespeichert wird?

Lösung

Dafür muss updateRow() aufgerufen werden.

Frage 11: Transaktionen

Was bewirkt setAutoCommit(false)?

Lösung

Danach werden SQL-Anweisungen nicht mehr automatisch sofort gespeichert. Änderungen müssen dann ausdrücklich mit commit() bestätigt werden.

Frage 12: commit und rollback

Was ist der Unterschied zwischen commit() und rollback()?

Lösung

commit() speichert die Änderungen dauerhaft.

rollback() macht die Änderungen der laufenden Transaktion wieder rückgängig.

Frage 13: Code verstehen

conn.setAutoCommit(false);

stmt1.executeUpdate();
stmt2.executeUpdate();

conn.commit();

Warum gehören diese Befehle logisch zusammen?

Lösung

Die beiden SQL-Anweisungen werden als eine gemeinsame Transaktion behandelt. Erst am Ende werden sie mit commit() zusammen gespeichert.

Frage 14: Batch Processing

Was ist die Grundidee von Batch Processing?

Lösung

Mehrere SQL-Anweisungen werden zuerst gesammelt und anschließend gemeinsam ausgeführt.

Frage 15: Batch-Methoden

Welche beiden Methoden sind für Batch Processing besonders wichtig?

Lösung

  • addBatch()
  • executeBatch()

Frage 16: Code verstehen

for (int i = 1; i <= 1000; i++) {
    stmt.setInt(1, i);
    stmt.setString(2, "Person_" + i);
    stmt.setInt(3, 20);
    stmt.addBatch();
}

stmt.executeBatch();

Was passiert hier?

Lösung

Es werden 1000 vorbereitete SQL-Anweisungen gesammelt und anschließend gemeinsam ausgeführt.

Frage 17: Architektur

Welche Aufgabe hat die Klasse Actor im Package domain?

Lösung

Sie beschreibt das Fachobjekt und speichert die Daten eines Schauspielers, zum Beispiel id, firstName und lastName.

Frage 18: Repository-Interface

Welche Aufgabe hat ein Repository-Interface wie ActorRepository?

Lösung

Das Interface beschreibt, welche Methoden verfügbar sind, ohne die konkrete Umsetzung festzulegen.

Frage 19: Architektur-Vorteil

Warum ist es sinnvoll, die Bereiche domain, persistence und app zu trennen?

Lösung

Dadurch wird der Code übersichtlicher, besser wartbar und leichter erweiterbar.

Außerdem kann man Implementierungen leichter austauschen, zum Beispiel für H2 oder SQL Server.

Frage 20: Optional

Was bedeutet Optional<Actor> bei der Methode findById(...)?

Lösung

Es bedeutet: Entweder wird ein Actor gefunden oder es gibt keinen Wert.

Frage 21: Fehler erkennen

Optional actor = repo.findById(1);
System.out.println(actor.get());

Was ist hier problematisch?

Lösung

Es wird ein roher Typ Optional verwendet statt Optional<Actor>.

Außerdem kann get() problematisch sein, wenn das Optional leer ist.

Frage 22: Code ergänzen

Ergänze die fehlende Methode, um alle Datensätze aus einem ResultSet nacheinander zu lesen: 

while (__________) {
    System.out.println(rs.getInt("id"));
}

Lösung

while (rs.next()) {
    System.out.println(rs.getInt("id"));
}

Frage 23: Code ergänzen

Ergänze die fehlende Methode für eine erfolgreiche Transaktion: 

conn.setAutoCommit(false);

stmt.executeUpdate();

conn.__________();

Lösung

conn.commit();

Frage 24: Gesamtverständnis

Warum sollte JDBC-Code nicht einfach komplett in der main-Methode stehen?

Lösung

Dann wird der Code schnell unübersichtlich.

Besser ist eine Trennung von Zuständigkeiten, zum Beispiel mit Domain-Klassen, Repository-Interface und JDBC-Implementierung.
