Ziel

Ziel ist es, ein scrollbares ResultSet zu verwenden und sich darin gezielt zu bewegen. Dabei sollen Methoden wie first(), last(), previous() und absolute(...) verwendet werden.

Aufgabe 6 – Scrollable ResultSet

Mit einem scrollbaren ResultSet navigieren

Aufgabe:

Erstelle ein Java-Programm, das eine Verbindung zu einer H2-Datenbank herstellt und Daten aus der Tabelle person ausliest.

Verwende dabei ein scrollbares ResultSet, sodass du dich nicht nur mit next() vorwärts bewegen kannst, sondern auch gezielt zum ersten, letzten oder zu einem bestimmten Datensatz springen kannst.

Gib jeweils den gefundenen Datensatz auf der Konsole aus.

Schritt 1: Properties-Datei verwenden

Verwende wieder eine Datei connection.properties mit den Verbindungsdaten für H2.

Hinweis

Du kannst dieselbe Properties-Datei wie in den vorigen Aufgaben verwenden.

Lösung: connection.properties

jdbc_url=jdbc:h2:mem:testdb;INIT=RUNSCRIPT FROM 'schema.sql'
jdbc_user=sa
jdbc_pwd=

Schritt 2: SQL-Datei erstellen

Lege eine Datei schema.sql an. Diese Datei soll eine Tabelle person erstellen und einige Beispieldaten einfügen.

Hinweis

Verwende wieder die Spalten id, name und age.

Lösung: schema.sql

DROP TABLE IF EXISTS person;

CREATE TABLE person (
    id INT PRIMARY KEY,
    name VARCHAR(100),
    age INT
);

INSERT INTO person VALUES (1, 'Anna', 22);
INSERT INTO person VALUES (2, 'Max', 25);
INSERT INTO person VALUES (3, 'Lisa', 19);
INSERT INTO person VALUES (4, 'Tom', 30);

Schritt 3: Verbindung herstellen

Lade die Properties-Datei und stelle eine Verbindung zur Datenbank her.

Hinweis

Verwende wieder Files.newInputStream(Path.of("connection.properties")) und DriverManager.getConnection(...).

Schritt 4: Scrollbares Statement erstellen

Erstelle ein Statement, das ein scrollbares ResultSet erzeugt.

Hinweis

Beim Erstellen des Statements kannst du zusätzlich den Typ des ResultSet angeben. 

Statement stmt = conn.createStatement(
    ResultSet.TYPE_SCROLL_INSENSITIVE,
    ResultSet.CONCUR_READ_ONLY
);

TYPE_SCROLL_INSENSITIVE bedeutet, dass du dich im ResultSet vor- und zurückbewegen kannst.

Schritt 5: Abfrage ausführen

Führe eine SELECT-Abfrage auf der Tabelle person aus.

Hinweis

Verwende zum Beispiel: 

SELECT * FROM person

Speichere das Ergebnis in einem ResultSet. 

ResultSet rs = stmt.executeQuery(sql);

Schritt 6: Zum ersten Datensatz springen

Springe gezielt zum ersten Datensatz und gib ihn aus.

Hinweis

Verwende: 

if (rs.first()) {
    System.out.println(rs.getInt("id"));
}

Mit first() springst du direkt zum ersten Datensatz.

Schritt 7: Zum letzten Datensatz springen

Springe gezielt zum letzten Datensatz und gib ihn aus.

Hinweis

Verwende: 

if (rs.last()) {
    System.out.println(rs.getString("name"));
}

Mit last() springst du direkt zum letzten Datensatz.

Schritt 8: Zu einem bestimmten Datensatz springen

Springe gezielt zum zweiten oder dritten Datensatz und gib ihn aus.

Hinweis

Verwende absolute(...): 

if (rs.absolute(2)) {
    System.out.println(rs.getString("name"));
}

absolute(2) bedeutet: gehe zum zweiten Datensatz.

Schritt 9: Einen Schritt zurückgehen

Bewege dich vom aktuellen Datensatz einen Schritt zurück und gib ihn aus.

Hinweis

Verwende: 

if (rs.previous()) {
    System.out.println(rs.getString("name"));
}

Mit previous() gehst du einen Datensatz zurück.

Erklärung

Standardmäßig wird ein ResultSet oft nur vorwärts gelesen. Dann kannst du nur mit next() arbeiten.

Ein scrollbares ResultSet erlaubt zusätzlich:

  • first() → zum ersten Datensatz
  • last() → zum letzten Datensatz
  • previous() → einen Datensatz zurück
  • absolute(n) → direkt zu Datensatz n

Das ist nützlich, wenn man sich gezielt im Ergebnis bewegen will.

Lösung

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.ResultSet;
import java.sql.SQLException;
import java.sql.Statement;
import java.io.InputStream;
import java.nio.file.Files;
import java.nio.file.Path;
import java.util.Properties;

public class JdbcScrollableResultSetExample {

    public static void main(String[] args) {

        Properties props = new Properties();
        Path path = Path.of("connection.properties");

        if (!Files.exists(path)) {
            System.out.println("Properties-Datei nicht gefunden!");
            return;
        }

        try (InputStream is = Files.newInputStream(path)) {

            props.load(is);

            String url = props.getProperty("jdbc_url");
            String user = props.getProperty("jdbc_user");
            String pwd = props.getProperty("jdbc_pwd");

            try (Connection conn = DriverManager.getConnection(url, user, pwd);
                 Statement stmt = conn.createStatement(
                         ResultSet.TYPE_SCROLL_INSENSITIVE,
                         ResultSet.CONCUR_READ_ONLY)) {

                String sql = "SELECT * FROM person";
                ResultSet rs = stmt.executeQuery(sql);

                if (rs.first()) {
                    System.out.println("Erster Datensatz:");
                    printPerson(rs);
                }

                if (rs.last()) {
                    System.out.println("Letzter Datensatz:");
                    printPerson(rs);
                }

                if (rs.absolute(2)) {
                    System.out.println("Zweiter Datensatz:");
                    printPerson(rs);
                }

                if (rs.previous()) {
                    System.out.println("Ein Datensatz zurück:");
                    printPerson(rs);
                }

            } catch (SQLException e) {
                System.out.println("Fehler bei der Datenbankoperation:");
                e.printStackTrace();
            }

        } catch (Exception e) {
            System.out.println("Fehler beim Laden der Properties:");
            e.printStackTrace();
        }
    }

    private static void printPerson(ResultSet rs) throws SQLException {
        int id = rs.getInt("id");
        String name = rs.getString("name");
        int age = rs.getInt("age");

        System.out.println(id + " | " + name + " | " + age);
        System.out.println();
    }
}
