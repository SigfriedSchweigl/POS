Ziel

Ziel ist es, ein ResultSet zu verwenden, in dem man sich gezielt vorwärts und rückwärts bewegen kann. Zusätzlich soll gezeigt werden, wie man einen Datensatz direkt über ein updatable ResultSet ändern kann.

Aufgabe 6 – Scrollable und Updatable ResultSet

Im ResultSet navigieren und Daten ändern

Aufgabe:

Erstelle ein Java-Programm, das eine Verbindung zu einer H2-Datenbank herstellt und Daten aus der Tabelle person ausliest.

Verwende dabei ein scrollbares und änderbares ResultSet. Du sollst:

  • zum ersten Datensatz springen
  • zum letzten Datensatz springen
  • zu einem bestimmten Datensatz springen
  • einen Schritt zurückgehen
  • einen Datensatz direkt im ResultSet ändern

Gib die jeweils gefundenen oder geänderten Daten auf der Konsole aus.

Hinweis: Standardmäßig kannst du dich im ResultSet nur mit next() vorwärts bewegen.

Für diese Aufgabe musst du ein scrollbares ResultSet verwenden, damit du auch rückwärts gehen und gezielt springen kannst.

Schritt 1: Properties-Datei verwenden

Verwende wieder eine Datei connection.properties mit den Verbindungsdaten für H2.

Hinweis

Du kannst dieselbe Properties-Datei wie in den vorigen Aufgaben verwenden.

Lösung: connection.properties

jdbc_url=jdbc:h2:mem:testdb;INIT=RUNSCRIPT FROM 'schema.sql'
jdbc_user=sa
jdbc_pwd=

Schritt 2: SQL-Datei erstellen

Lege eine Datei schema.sql an. Diese Datei soll eine Tabelle person erstellen und einige Beispieldaten einfügen.

Hinweis

Verwende wieder die Spalten id, name und age.

Lösung: schema.sql

DROP TABLE IF EXISTS person;

CREATE TABLE person (
    id INT PRIMARY KEY,
    name VARCHAR(100),
    age INT
);

INSERT INTO person VALUES (1, 'Anna', 22);
INSERT INTO person VALUES (2, 'Max', 25);
INSERT INTO person VALUES (3, 'Lisa', 19);
INSERT INTO person VALUES (4, 'Tom', 30);

Schritt 3: Verbindung herstellen

Lade die Properties-Datei und stelle eine Verbindung zur Datenbank her.

Hinweis

Verwende wieder Files.newInputStream(Path.of("connection.properties")) und DriverManager.getConnection(...).

Schritt 4: Scrollbares und änderbares Statement erstellen

Erstelle ein Statement, das ein scrollbares und änderbares ResultSet erzeugt.

Hinweis

Standardmäßig ist ein ResultSet oft nur vorwärts lesbar (next()) und nicht direkt änderbar.

Damit du dich vorwärts und rückwärts bewegen und Daten ändern kannst, musst du das Statement speziell erstellen: 

Statement stmt = conn.createStatement(
    ResultSet.TYPE_SCROLL_INSENSITIVE,
    ResultSet.CONCUR_UPDATABLE
);

TYPE_SCROLL_INSENSITIVE → erlaubt Vor- und Zurückspringen
CONCUR_UPDATABLE → erlaubt Änderungen über das ResultSet

Schritt 5: Abfrage ausführen

Führe eine SELECT-Abfrage auf der Tabelle person aus.

Hinweis

Verwende zum Beispiel: 

SELECT * FROM person

Speichere das Ergebnis in einem ResultSet. 

ResultSet rs = stmt.executeQuery(sql);

Schritt 6: Zum ersten Datensatz springen

Springe direkt zum ersten Datensatz und gib ihn aus.

Hinweis

Springe direkt zum ersten Datensatz:

if (rs.first()) {
    int id = rs.getInt("id");
    String name = rs.getString("name");
}

first() positioniert den Cursor auf den ersten Datensatz.

Schritt 7: Zum letzten Datensatz springen

Springe direkt zum letzten Datensatz und gib ihn aus.

Hinweis

Verwende:

if (rs.last()) {
    System.out.println(rs.getString("name"));
}

Mit last() springst du direkt zum letzten Datensatz.

Schritt 8: Zu einem bestimmten Datensatz springen

Springe gezielt zu einem bestimmten Datensatz und gib ihn aus.

Hinweis

Mit absolute(n) springst du direkt zu einer bestimmten Position: 

rs.absolute(2);

→ bedeutet: gehe zum 2. Datensatz 

if (rs.absolute(2)) {
    System.out.println(rs.getString("name"));
}

Schritt 9: Einen Schritt zurückgehen

Bewege dich vom aktuellen Datensatz einen Schritt zurück und gib ihn aus.

Hinweis

Verwende:

if (rs.previous()) {
    System.out.println(rs.getString("name"));
}

Mit previous() gehst du einen Datensatz zurück.

Schritt 10: Datensatz direkt im ResultSet ändern

Ändere das Alter einer Person direkt über das ResultSet.

Hinweis

Springe zuerst zu einem Datensatz, zum Beispiel zum zweiten: 

rs.absolute(2);

Ändere dann eine Spalte: 

rs.updateInt("age", 99);

Übernimm die Änderung mit: 

rs.updateRow();

Schritt 11: Änderung überprüfen

Gib den geänderten Datensatz erneut aus.

Hinweis

Lies nach updateRow() die Werte erneut aus oder springe nochmals zu diesem Datensatz.

Erklärung

Ohne spezielle Einstellungen kann ein ResultSet oft nur vorwärts gelesen werden: 

rs.next();

Ein scrollbares ResultSet erlaubt zusätzliche Bewegungen:

  • first() → erster Datensatz
  • last() → letzter Datensatz
  • previous() → zurück
  • absolute(n) → direkt zu Position n

Ein updatable ResultSet erlaubt außerdem direkte Änderungen:

  • updateString(...)
  • updateInt(...)
  • updateRow()

👉 Der Cursor bewegt sich also frei innerhalb der Ergebnismenge und Änderungen können direkt auf den aktuellen Datensatz angewendet werden.

Lösung

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.ResultSet;
import java.sql.SQLException;
import java.sql.Statement;
import java.io.InputStream;
import java.nio.file.Files;
import java.nio.file.Path;
import java.util.Properties;

public class JdbcScrollableUpdatableResultSetExample {

    public static void main(String[] args) {

        Properties props = new Properties();
        Path path = Path.of("connection.properties");

        if (!Files.exists(path)) {
            System.out.println("Properties-Datei nicht gefunden!");
            return;
        }

        try (InputStream is = Files.newInputStream(path)) {

            props.load(is);

            String url = props.getProperty("jdbc_url");
            String user = props.getProperty("jdbc_user");
            String pwd = props.getProperty("jdbc_pwd");

            try (Connection conn = DriverManager.getConnection(url, user, pwd);
                 Statement stmt = conn.createStatement(
                         ResultSet.TYPE_SCROLL_INSENSITIVE,
                         ResultSet.CONCUR_UPDATABLE)) {

                String sql = "SELECT * FROM person";
                ResultSet rs = stmt.executeQuery(sql);

                if (rs.first()) {
                    System.out.println("Erster Datensatz:");
                    printPerson(rs);
                }

                if (rs.last()) {
                    System.out.println("Letzter Datensatz:");
                    printPerson(rs);
                }

                if (rs.absolute(2)) {
                    System.out.println("Zweiter Datensatz:");
                    printPerson(rs);
                }

                if (rs.previous()) {
                    System.out.println("Ein Datensatz zurück:");
                    printPerson(rs);
                }

                if (rs.absolute(2)) {
                    System.out.println("Datensatz vor der Änderung:");
                    printPerson(rs);

                    rs.updateInt("age", 99);
                    rs.updateRow();

                    System.out.println("Datensatz nach der Änderung:");
                    printPerson(rs);
                }

            } catch (SQLException e) {
                System.out.println("Fehler bei der Datenbankoperation:");
                e.printStackTrace();
            }

        } catch (Exception e) {
            System.out.println("Fehler beim Laden der Properties:");
            e.printStackTrace();
        }
    }

    private static void printPerson(ResultSet rs) throws SQLException {
        int id = rs.getInt("id");
        String name = rs.getString("name");
        int age = rs.getInt("age");

        System.out.println(id + " | " + name + " | " + age);
        System.out.println();
    }
}
