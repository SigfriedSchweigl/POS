Ziel

Aufgabe 4 – H2 mit PreparedStatement

Daten sicher mit PreparedStatement abfragen

Aufgabe:

Erstelle ein Java-Programm, das die Verbindungsdaten aus einer connection.properties-Datei lädt und damit eine Verbindung zu einer H2-Datenbank herstellt.

Über die JDBC-URL soll beim Verbindungsaufbau automatisch ein SQL-Skript ausgeführt werden. Dieses Skript soll eine Tabelle erstellen und Beispieldaten einfügen.

Suche anschließend mit einem PreparedStatement gezielt nach einer Person anhand ihres Namens und gib die gefundenen Daten auf der Konsole aus.

Vergleiche außerdem eine unsichere SQL-Abfrage mit String-Verkettung mit einer sicheren Variante mit Platzhaltern.

Schritt 1: Properties-Datei verwenden

Verwende wieder eine Datei connection.properties mit den Verbindungsdaten für H2.

Hinweis

Du kannst dieselbe Properties-Datei wie in den vorigen Aufgaben verwenden.

Lösung: connection.properties

jdbc_url=jdbc:h2:mem:testdb;INIT=RUNSCRIPT FROM 'schema.sql'
jdbc_user=sa
jdbc_pwd=

Schritt 2: SQL-Datei erstellen

Lege eine Datei schema.sql an. Diese Datei soll eine Tabelle person erstellen und einige Beispieldaten einfügen.

Hinweis

Verwende wieder die Spalten id, name und age.

Lösung: schema.sql

DROP TABLE IF EXISTS person;

CREATE TABLE person (
    id INT PRIMARY KEY,
    name VARCHAR(100),
    age INT
);

INSERT INTO person VALUES (1, 'Anna', 22);
INSERT INTO person VALUES (2, 'Max', 25);
INSERT INTO person VALUES (3, 'Lisa', 19);
INSERT INTO person VALUES (4, 'Tom', 30);

Schritt 3: Verbindung herstellen

Lade die Properties-Datei und stelle eine Verbindung zur Datenbank her.

Hinweis

Verwende wieder Files.newInputStream(Path.of("connection.properties")) und DriverManager.getConnection(...).

Schritt 4: PreparedStatement erstellen

Erstelle ein PreparedStatement mit einem Platzhalter für den Namen.

Hinweis

Erstelle zuerst eine SQL-Abfrage mit einem Platzhalter: 

SELECT * FROM person WHERE name = ?

Verwende diese Abfrage, um ein PreparedStatement zu erzeugen: 

PreparedStatement pstmt = conn.prepareStatement(sql);

Speichere die SQL-Anweisung dazu vorher in einer String-Variable.

Schritt 5: Parameter setzen

Weise dem Platzhalter einen konkreten Namen zu, zum Beispiel Lisa.

Hinweis

Setze den Wert für den Platzhalter mit einer passenden Methode: 

pstmt.setString(1, "Lisa");

Die Zahl 1 steht für den ersten Platzhalter ?.

Schritt 6: Abfrage ausführen

Führe die Abfrage mit executeQuery() aus und lies das ResultSet aus.

Hinweis

Führe die Abfrage mit dem PreparedStatement aus: 

ResultSet rs = pstmt.executeQuery();

Durchlaufe anschließend das Ergebnis: 

while (rs.next()) {
      String name = rs.getString("name");
  }

Schritt 7: Unsichere Variante verstehen

Betrachte zusätzlich eine unsichere Abfrage, bei der ein Benutzerwert direkt in den SQL-String eingefügt wird.

Hinweis

Bei der unsicheren Variante wird der Benutzerwert direkt in den SQL-String eingebaut. 

String input = "' OR '1'='1";
  String sql = "SELECT * FROM person WHERE name = '" + input + "'";

Diese SQL-Anweisung wird anschließend mit einem normalen Statement ausgeführt: 

Statement stmt = conn.createStatement();
  ResultSet rs = stmt.executeQuery(sql);

So kann man sehen, warum String-Verkettung problematisch ist und warum PreparedStatement sicherer ist.

Erklärung

Bei einer String-Verkettung kann Benutzereingabe die SQL-Anweisung verändern. Dadurch kann es passieren, dass eine Abfrage mehr Daten zurückliefert als beabsichtigt.

Beispiel für eine problematische Eingabe: 

' OR '1'='1

Diese Eingabe kann eine WHERE-Bedingung so verändern, dass sie immer wahr ist.

Ein PreparedStatement behandelt Eingaben als Werte und nicht als Teil des SQL-Codes. Dadurch wird die Abfrage sicherer und besser lesbar.

Schritt 8: Sichere Variante ausführen

Führe dieselbe Suche mit PreparedStatement aus und beobachte, dass die Eingabe als normaler Wert behandelt wird.

Hinweis

Mit PreparedStatement bleibt die SQL-Struktur unverändert, auch wenn die Eingabe Sonderzeichen enthält.

Lösung

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.PreparedStatement;
import java.sql.ResultSet;
import java.sql.SQLException;
import java.sql.Statement;
import java.io.InputStream;
import java.nio.file.Files;
import java.nio.file.Path;
import java.util.Properties;

public class JdbcPreparedStatementExample {

    public static void main(String[] args) {

        Properties props = new Properties();
        Path path = Path.of("connection.properties");

        if (!Files.exists(path)) {
            System.out.println("Properties-Datei nicht gefunden!");
            return;
        }

        try (InputStream is = Files.newInputStream(path)) {

            props.load(is);

            String url = props.getProperty("jdbc_url");
            String user = props.getProperty("jdbc_user");
            String pwd = props.getProperty("jdbc_pwd");

            try (Connection conn = DriverManager.getConnection(url, user, pwd)) {

                System.out.println("Verbindung erfolgreich!");

                String searchedName = "Lisa";

                String safeSql = "SELECT * FROM person WHERE name = ?";

                try (PreparedStatement pstmt = conn.prepareStatement(safeSql)) {
                    pstmt.setString(1, searchedName);

                    ResultSet rs = pstmt.executeQuery();

                    System.out.println("Sichere Suche nach: " + searchedName);

                    while (rs.next()) {
                        int id = rs.getInt("id");
                        String name = rs.getString("name");
                        int age = rs.getInt("age");

                        System.out.println(id + " | " + name + " | " + age);
                    }
                }

                System.out.println();
                System.out.println("--- Unsicheres Beispiel mit String-Verkettung ---");

                String userInput = "' OR '1'='1";
                String unsafeSql =
                        "SELECT * FROM person WHERE name = '" + userInput + "'";

                System.out.println("Unsichere SQL-Anweisung:");
                System.out.println(unsafeSql);

                try (Statement stmt = conn.createStatement()) {
                    ResultSet rs = stmt.executeQuery(unsafeSql);

                    while (rs.next()) {
                        int id = rs.getInt("id");
                        String name = rs.getString("name");
                        int age = rs.getInt("age");

                        System.out.println(id + " | " + name + " | " + age);
                    }
                }

                System.out.println();
                System.out.println("--- Sichere Variante mit derselben Eingabe ---");

                try (PreparedStatement pstmt = conn.prepareStatement(
                        "SELECT * FROM person WHERE name = ?")) {

                    pstmt.setString(1, userInput);

                    ResultSet rs = pstmt.executeQuery();

                    if (!rs.next()) {
                        System.out.println("Keine Person gefunden.");
                    } else {
                        do {
                            int id = rs.getInt("id");
                            String name = rs.getString("name");
                            int age = rs.getInt("age");

                            System.out.println(id + " | " + name + " | " + age);
                        } while (rs.next());
                    }
                }

            } catch (SQLException e) {
                System.out.println("Fehler bei der Datenbankoperation:");
                e.printStackTrace();
            }

        } catch (Exception e) {
            System.out.println("Fehler beim Laden der Properties:");
            e.printStackTrace();
        }
    }
}
